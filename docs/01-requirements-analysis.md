# 健身助手需求分析

## 1. 角色

| 角色 | 描述 | 核心诉求 |
| --- | --- | --- |
| 使用者 | 健身新手或恢复训练者，当前身高 174cm，体重 142 斤/71kg | 减脂、瘦肚子、瘦腿、练腹肌，重点训练肩、背、肱二头、肱三头 |
| 健身助手 | 系统中的训练计划和训练记录模块 | 管理训练计划、记录训练完成情况、输出训练统计 |
| 营养助手 | 系统中的饮食记录和营养分析模块 | 记录每日饮食，估算热量和三大营养素，给出饮食建议 |
| 数据统计后台 | 系统中的统计分析模块 | 汇总打卡、训练、饮食、体重、腰围等趋势 |

## 2. 背景

| 项目 | 内容 |
| --- | --- |
| 当前日期 | 2026-07-06 |
| 计划周期 | 2026-07-06 到 2026-08-26 |
| 当前状态 | 健身新手，中断运动约 1-2 周，需要先恢复再加量 |
| 训练频率 | 每周 3-4 次 |
| 训练目标 | 减脂、瘦肚子、瘦腿、练腹肌，强化肩、背、肱二头、肱三头 |
| 家中器械 | 小哑铃、瑜伽垫、臂力棒 |
| 常见食材 | 大米、小米、八宝粥、面包、馒头、花卷、猪肉、鱼肉、鸡蛋、土豆、茄子、辣椒、蔬菜、水果、西瓜等 |

## 3. 业务目标

| 目标 | 说明 |
| --- | --- |
| 计划可视化 | 用表格、卡片、图表展示完整 7 周半计划 |
| 数据可记录 | 每日训练、饮食、体重、腰围、步数、睡眠都能录入 |
| 数据可保存 | 所有用户录入内容必须保存到本地 SQLite 数据库 |
| 数据可分析 | 根据记录计算完成率、连续打卡、热量趋势、体重腰围变化 |
| 计划可调整 | 用户可以新增、修改、删除自定义任务 |
| 饮食可评估 | 输入日常食物后，系统估算热量和营养结构 |
| 后续可扩展 | 保留食物库、训练库、AI 建议、数据导出等扩展空间 |

### 4.4 饮食记录与分析

用户可以按餐次输入：

- 早餐
- 午餐
- 晚餐
- 加餐

系统需要估算：

| 指标 | 说明 |
| --- | --- |
| 热量 | kcal，按食物库和份量估算 |
| 蛋白质 | g，估算蛋白质摄入 |
| 碳水 | g，估算主食和糖分摄入 |
| 脂肪 | g，估算油脂和肉类脂肪 |
| 饮食建议 | 根据总热量、蛋白质、碳水、晚餐结构给出短建议 |

第一版允许用户用自然语言输入，例如：

```text
早餐：小米粥一碗，鸡蛋两个，面包一片
午餐：米饭一碗，土豆炒鸡蛋，炖鱼
晚餐：馒头半个，茄子，鸡蛋一个
```

系统可先采用关键词匹配和默认份量估算，不要求医学级精确。

## 5. 边界条件

| 边界 | 说明 |
| --- | --- |
| 数据库要求 | 必须使用本地 SQLite，不能只用 localStorage |
| 运行方式 | 本地前端页面加本地后端服务 |
| 用户范围 | 第一版只支持单用户，不做登录注册 |
| 网络依赖 | 第一版不依赖云服务 |
| 营养分析 | 第一版为估算，不作为医疗建议 |
| 训练建议 | 默认面向健康成年人，若有伤病需用户自行调整或咨询医生 |
| 数据真实性 | 统计只基于用户实际输入，不虚构缺失数据 |
| 隐私 | 数据默认保存在本机，不上传云端 |

## 6. 专业扩展建议

### 6.1 疲劳和恢复管理

加入 RPE、睡眠、酸痛程度记录。系统根据疲劳程度提醒减少训练量、改为拉伸或快走，降低新手过度训练风险。

### 6.2 食材模板和购物清单

根据常见食材生成减脂餐模板，例如鱼肉加青菜加半碗米饭、鸡蛋加小米粥加黄瓜。后续可自动生成每周购物清单。

### 6.3 身体围度和阶段复盘

除体重外，加入腰围、大腿围、肩围、照片备注等记录。减脂期体重可能波动，围度变化更能反映效果。
