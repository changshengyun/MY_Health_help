# 多 Agent 工作流与总体验收

## 1. 总 A 职责

总 A 是本轮扩展开发的任务分发者、接口协调者和最终验收者，负责把用户需求拆成可并行执行的 Agent 任务，并在各 Agent 交付后进行统一验收。

总 A 的核心职责：

- 明确本轮目标、边界和不可修改范围，避免不同 Agent 之间重复开发或互相覆盖。
- 将 4 个专业扩展点拆分给 A1-A4，并说明每个 Agent 的输入、输出和验收标准。
- 维护跨 Agent 的一致性，包括数据命名、页面入口、交互文案、接口契约和文档口径。
- 在每个 Agent 完成后检查交付物是否满足需求、是否影响其他模块、是否存在明显回归。
- 汇总最终验收清单，确认训练恢复、餐食模板/购物清单、身体围度目标、阶段复盘/教练建议 4 个扩展点均可被识别、使用和验证。
- 不直接回滚其他 Agent 或用户的修改。如发现冲突，应先记录冲突点，再由总 A 协调对应 Agent 修正。

## 2. A1-A4 分工

本轮采用“总 A 分发任务 + 多 Agent 分组开发 + 总 A 验收”的工作流。

- A1：训练恢复扩展点。
- A2：餐食模板/购物清单扩展点。
- A3：身体围度目标扩展点。
- A4：总体验收与多 Agent 工作流文档。

各 Agent 应在自己的任务边界内完成实现、说明和自测，不主动修改其他 Agent 的职责范围。若实现需要依赖其他 Agent 的数据结构、页面入口或接口，应在交付说明中明确依赖关系。

## 3. 每个 Agent 的输入、输出、验收标准

### A1：训练恢复

输入：

- 用户关于训练计划、训练后反馈、疲劳状态、伤痛或恢复需求的扩展要求。
- 现有健康助手项目中的训练记录、计划展示、建议生成或相关数据结构。
- 总 A 给出的任务边界和本轮 4 个扩展点清单。

输出：

- 训练恢复相关功能或数据说明。
- 可识别用户训练后状态的字段、交互或展示入口。
- 面向用户的恢复建议内容，包括强度调整、休息建议、拉伸或恢复重点。
- 必要的实现说明和自测结果。

验收标准：

- 明确支持“训练恢复”这一扩展点，而不是只保留普通训练计划。
- 能表达训练完成后的恢复状态，例如疲劳、酸痛、恢复日、降低强度或暂停训练。
- 恢复建议与用户训练状态相关，不能是完全静态且无上下文的泛化文本。
- 不破坏既有训练计划、记录或健康助手主流程。
- 交付内容包含可由总 A 检查的入口、数据字段或展示结果。

### A2：餐食模板/购物清单

输入：

- 用户关于饮食计划、餐食模板、食材准备和购物清单的扩展要求。
- 现有项目中的饮食、营养、计划或建议相关模块。
- A1、A3、A4 的交付边界，以及总 A 对统一文案和数据口径的要求。

输出：

- 餐食模板功能或文档说明。
- 从餐食模板生成购物清单的规则、数据结构或用户界面入口。
- 面向用户的餐次、食材、数量、分类或替换建议。
- 必要的实现说明和自测结果。

验收标准：

- 明确支持“餐食模板/购物清单”扩展点。
- 餐食模板至少能表达餐次、主要食材和营养或目标倾向。
- 购物清单能从餐食模板中被推导或关联，而不是与模板完全割裂。
- 展示内容对用户可执行，例如能看出需要购买哪些食材。
- 不影响既有饮食建议或健康助手主流程。

### A3：身体围度目标

输入：

- 用户关于身体围度、体型目标、阶段目标和追踪指标的扩展要求。
- 现有项目中的个人档案、身体指标、目标管理或进度展示模块。
- 总 A 提供的本轮扩展点清单和跨模块一致性要求。

输出：

- 身体围度目标相关功能或文档说明。
- 可记录或展示的围度指标，例如腰围、臀围、胸围、臂围、腿围等。
- 目标值、当前值、变化趋势或阶段进度的表达方式。
- 必要的实现说明和自测结果。

验收标准：

- 明确支持“身体围度目标”扩展点。
- 围度目标与体重、训练或饮食目标有清晰区分，不能只复用体重目标。
- 至少支持当前值和目标值，最好支持阶段性变化或进度展示。
- 指标命名、单位和展示文案清楚，用户能理解如何记录和比较。
- 不破坏既有身体指标、个人档案或目标管理流程。

### A4：总体验收与多 Agent 工作流文档

输入：

- 用户对多 Agent 工作流文档的明确要求。
- 本轮 4 个扩展点：训练恢复、餐食模板/购物清单、身体围度目标、阶段复盘/教练建议。
- 总 A 分发、A1-A4 分工、总 A 验收的协作模式。
- 当前项目路径和写入范围限制。

输出：

- `docs/05-agent-workflow-acceptance.md`。
- 文档中覆盖总 A 职责、A1-A4 分工、每个 Agent 的输入/输出/验收标准、4 个扩展点说明、总体验收清单。

验收标准：

- 文档文件存在，路径正确。
- 文档为 UTF-8，中文内容正常。
- 明确写出 4 个专业扩展点。
- 明确每个 Agent 的输入、输出和验收标准。
- 只新增或修改 `docs/05-agent-workflow-acceptance.md`，不修改其他文件。

## 4. 4 个专业扩展点说明

### 4.1 训练恢复

训练恢复关注用户完成训练后的身体反馈和下一步调整。它不是单纯生成训练计划，而是根据疲劳、酸痛、训练完成度、恢复日安排等信息，帮助用户判断是否需要降低强度、增加休息、调整动作或加入拉伸恢复。

该扩展点应让用户看到训练后的恢复建议，并能区分“继续推进训练”和“需要恢复调整”的不同场景。

### 4.2 餐食模板/购物清单

餐食模板用于沉淀可复用的饮食方案，例如减脂早餐、高蛋白午餐、训练日晚餐或便捷加餐。购物清单则根据模板中的食材和份量整理出用户需要准备或购买的内容。

该扩展点应解决“知道吃什么”到“知道买什么”的衔接问题，输出应具备可执行性。

### 4.3 身体围度目标

身体围度目标用于追踪体型变化，例如腰围、臀围、胸围、臂围、腿围等。它补充了体重指标无法完整表达身体变化的问题，适合与训练、饮食和阶段目标共同使用。

该扩展点应支持当前围度、目标围度和进度对比，让用户能用更细的指标判断身体变化。

### 4.4 阶段复盘/教练建议

阶段复盘/教练建议用于在一段时间后总结用户执行情况，并给出下一阶段调整方向。复盘内容可以覆盖训练完成度、饮食执行、身体指标变化、围度变化、恢复状态和目标偏差。

该扩展点应像教练一样给出结构化判断，包括做得好的地方、主要问题、调整建议和下一阶段重点。

## 5. 总体验收清单

总 A 在最终验收时应逐项检查：

- 文档与实现均能明确识别 4 个扩展点：训练恢复、餐食模板/购物清单、身体围度目标、阶段复盘/教练建议。
- A1 的训练恢复交付有清晰入口、数据表达或建议输出。
- A2 的餐食模板/购物清单交付能从模板关联到可执行购物清单。
- A3 的身体围度目标交付能表达当前值、目标值和围度类指标。
- A4 的工作流与验收文档存在，且覆盖总 A 职责、A1-A4 分工和每个 Agent 的验收标准。
- 各 Agent 的输出边界清楚，没有无关的大范围重构。
- 各 Agent 未回滚或覆盖其他开发者修改。
- 关键用户文案为正常中文，无乱码、错别字集中出现或明显不可读内容。
- 新增功能与既有健康助手主流程兼容，不造成页面、接口或数据加载明显回归。
- 如存在跨 Agent 依赖，依赖关系已在交付说明或文档中说明。

## 6. 总 A 验收流程

1. 收集 A1-A4 的交付说明，确认每个 Agent 都说明了输入、输出和自测结果。
2. 对照本文件的每个 Agent 验收标准，检查交付物是否满足最小可验收要求。
3. 重点检查 4 个专业扩展点是否都能被用户看见、理解或使用。
4. 检查 Git 工作区，确认没有超出任务范围的意外文件变更。
5. 如发现问题，按责任边界退回对应 Agent 修正；如问题来自跨 Agent 契约不一致，由总 A 统一协调。
6. 全部通过后，总 A 汇总最终验收结论，并说明已验证的扩展点、剩余风险和后续建议。
