# 后续开发计划

## 1. 开发原则

- 先做真实可用闭环，再做复杂智能分析。
- 所有新增、修改、删除都通过后端 API 写入 SQLite。
- 页面优先清晰、耐用、适合每天打开使用，不做纯展示型落地页。
- 统计必须基于真实记录，缺失数据不补造。
- 第一版不做登录、多用户、云同步。

## 4. 开发顺序建议

1. 后端健康检查和 SQLite 连接。
2. 数据库 schema 和默认 seed 数据。
3. 前端总布局和导航。
4. 分周计划静态数据改为 API 数据。
5. 每日打卡真实保存。
6. 饮食记录和热量估算。
7. 训练记录。
8. 统计图表。
9. UI 打磨和验收。

## 5. 第一版非目标

| 暂不开发 | 原因 |
| --- | --- |
| 登录注册 | 当前只给个人本地使用 |
| 云同步 | 增加隐私和部署复杂度 |
| 手机 App | 先完成 Web 本地版 |
| 医学级营养分析 | 第一版只做估算 |
| 图像识别食物 | 需要额外模型和复杂度 |

## 6. 后续版本方向

| 版本 | 方向 |
| --- | --- |
| V1 | 本地可用、完整记录和统计 |
| V1.1 | 食物库编辑、餐食模板、购物清单 |
| V1.2 | 疲劳恢复建议、训练量自动调整 |
| V1.3 | 数据导出 CSV/Excel、阶段报告 |
| V2 | 可选 AI 饮食分析和训练建议 |
